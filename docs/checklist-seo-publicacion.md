# Checklist SEO de Publicacion

Usar esta lista cada vez que se agregue una pagina nueva o se hagan cambios importantes.

## 1) Antes de publicar

- [ ] La pagina tiene `title` y `meta description` claros.
- [ ] La pagina tiene `canonical` apuntando a su URL final (`https://piceadigital.com/...`).
- [ ] Los enlaces internos usan la version canonica (evitar mezclar `http`, `https`, `www` y no-`www`).
- [ ] La URL nueva esta enlazada desde alguna pagina indexable (home, portfolio o menu).
- [ ] Si aplica, la pagina incluye datos estructurados (`schema.org`).

## 2) Sitemap y robots

- [ ] Agregar la URL nueva en `sitemap.xml`.
- [ ] Actualizar `lastmod` de las URLs editadas en `sitemap.xml`.
- [ ] Verificar que `robots.txt` siga apuntando a `https://piceadigital.com/sitemap.xml`.
- [ ] No incluir URLs de pruebas o demos (por ejemplo `/temp/`) en el sitemap.

## 3) Search Console

- [ ] Ir a `Sitemaps` y confirmar estado `Correcto`.
- [ ] Inspeccionar la URL nueva en `Inspeccion de URLs`.
- [ ] Hacer clic en `Solicitar indexacion`.
- [ ] Revisar `Paginas` por posibles errores de indexacion.

## 4) Verificacion final

- [ ] Probar la URL publicada en navegador (desktop y mobile).
- [ ] Comprobar que no haya errores en formulario, botones de WhatsApp y enlaces clave.
- [ ] Confirmar que meta y snippet se ven coherentes al compartir (Open Graph/Twitter si aplica).

## 5) Control semanal (recomendado)

- [ ] Revisar `Rendimiento` en Search Console (clics, impresiones, CTR, posicion).
- [ ] Detectar consultas nuevas y ajustar copy/titulos si hace falta.
- [ ] Revisar URLs no indexadas y validar correcciones.
